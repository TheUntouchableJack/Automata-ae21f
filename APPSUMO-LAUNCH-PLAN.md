## Part 4: The Product Changes

### New Database Tables Needed

```sql
-- AI Actions Log (track what AI does automatically)
CREATE TABLE ai_actions_log (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    organization_id UUID REFERENCES organizations(id),
    action_type TEXT NOT NULL, -- 'win_back_sent', 'birthday_reward', 'tier_upgrade', etc.
    member_id UUID REFERENCES app_members(id),
    description TEXT,
    result TEXT, -- 'sent', 'opened', 'converted', etc.
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Automated Campaigns (what AI can trigger)
CREATE TABLE automated_campaigns (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    app_id UUID REFERENCES customer_apps(id),
    campaign_type TEXT NOT NULL, -- 'win_back', 'birthday', 'milestone', 'referral_nudge'
    is_enabled BOOLEAN DEFAULT true,
    settings JSONB DEFAULT '{}', -- thresholds, message templates, etc.
    last_run_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Digital Punch Cards
CREATE TABLE punch_cards (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    app_id UUID REFERENCES customer_apps(id),
    name TEXT NOT NULL,
    punches_required INTEGER NOT NULL DEFAULT 10,
    reward_description TEXT,
    is_active BOOLEAN DEFAULT true,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE member_punch_cards (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    member_id UUID REFERENCES app_members(id),
    punch_card_id UUID REFERENCES punch_cards(id),
    punches_collected INTEGER DEFAULT 0,
    completed_at TIMESTAMPTZ,
    reward_redeemed_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

## Part 9: FAQ (For Launch)

**Q: Does my customer need to download an app?**
A: No. Your loyalty app works in any mobile browser. Customers just scan a QR code—no download required.

**Q: How is this different from Stamp Me or Square Loyalty?**
A: Those tools let you SET UP a loyalty program. Royalty's AI RUNS it for you—analyzing customer behavior and telling you exactly what to do to grow. It's like having a marketing manager built in.

**Q: What if I already have a loyalty program?**
A: You can import existing customers and their points. We'll also analyze your data and immediately show you opportunities you're missing.

**Q: Does the AI actually work?**
A: Yes. It analyzes real patterns—visit frequency, spending, timing—and generates specific, actionable recommendations. It's not generic tips; it's "Customer Sarah hasn't visited in 14 days, here's a win-back offer ready to send."

**Q: How do customers earn points?**
A: They scan a QR code at your location. You can also award points manually for purchases. No POS integration required (though we support it).

**Q: Can I customize the branding?**
A: Yes. Your logo, colors, and business info. Higher tiers include full white-label (your domain, no Royalty branding).

**Q: What languages are supported?**
A: English, Spanish, French, German, Italian, Portuguese, Chinese, and Arabic. Your customers can use the app in their preferred language.

**Q: Is there a limit on customers?**
A: Tier 1: 500 members. Tier 2: 2,000 members. Tier 3: Unlimited.

**Q: What happens after the lifetime deal ends?**
A: Nothing—it's lifetime. You keep all features from your tier forever. Future features may require an upgrade, but core functionality is locked in.

**Q: Do you have an API?**
A: Yes, on Tier 3. You can integrate with your existing systems.
