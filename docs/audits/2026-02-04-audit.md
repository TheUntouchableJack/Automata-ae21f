## Critical Issues (Fix Immediately)

### 1. [SECURITY] XSS via `document.write()` in Print Functions

**Files:** [dashboard.js:566-580](app/dashboard.js#L566-L580), [app-builder.js:1199-1218](app/app-builder.js#L1199-L1218)

**Risk:** `printPreviewQR()` and `printQRCode()` inject `currentApp.name` directly into HTML via `document.write()` without escaping. A business name containing `<script>alert(1)</script>` would execute in the print window.

**Fix:** Use `escapeHtml()` on all interpolated values in `document.write()` calls:
```javascript
// dashboard.js:570 â€” change:
<h2 style="margin-bottom: 8px;">${currentApp?.name || 'My App'}</h2>
// to:
<h2 style="margin-bottom: 8px;">${escapeHtml(currentApp?.name || 'My App')}</h2>
```
Same pattern for `app-builder.js:1207` and the URL display on line 1210.

### 2. [SECURITY] `showPersistentError()` Renders Unsanitized HTML

**File:** [app-builder.js:1297](app/app-builder.js#L1297)

**Risk:** `showPersistentError(message, helpLink)` injects `message` directly into `innerHTML`. If an error message from Supabase contains HTML, it would be rendered. While the current callers pass hardcoded strings, this is a latent injection vector.

**Fix:** Use `textContent` instead of innerHTML for the message span, or pass through `escapeHtml()`:
```javascript
<span>${escapeHtml(message)}</span>
```

## High Priority (Fix Before Launch)

### 3. [QA] Duplicate Event Listeners on Preview Panel

**File:** [dashboard.js:404-458](app/dashboard.js#L404-L458)

**Risk:** `showPreviewPanel()` adds `addEventListener` on every call without removing old listeners. If the function is called multiple times (e.g., dashboard refresh), click handlers stack up â€” causing duplicate `window.open()` calls, duplicate clipboard writes, etc.

**Fix:** Use `{ once: true }`, extract named handler functions with `removeEventListener`, or add a guard flag:
```javascript
if (previewCol.dataset.initialized) return;
previewCol.dataset.initialized = 'true';
```

### 4. [CODE] `generateSlug()` Duplicated Across Files

**Files:** [dashboard.js:356-362](app/dashboard.js#L356-L362), [app-builder.js:1262-1268](app/app-builder.js#L1262-L1268)

**Risk:** Two identical implementations of the same function. If one is updated (e.g., max length change), the other won't match, causing slug inconsistencies.

**Fix:** Move to `utils.js` and reference `AppUtils.generateSlug()` in both files.

### 5. [DESIGN] Hardcoded English Strings in Dashboard

**File:** [dashboard.js:400](app/dashboard.js#L400)

**Risk:** `"Rewards Program"` subtitle in the preview splash is hardcoded, not i18n. Also, `getTimeAgo()` returns English time labels ("just now", "5m ago"), and chart date formatting uses `en-US` locale.

**Fix:**
- Add `dashboard.previewSubtitle` i18n key for "Rewards Program"
- Use `i18n.t()` for time-ago strings or create locale-aware helper
- Use `navigator.language` or user's locale preference for date formatting

### 6. [ARCHITECTURE] QR Fallback Leaks App URLs to Third-Party API

**Files:** [dashboard.js:527-528](app/dashboard.js#L527-L528), [app-builder.js:1053-1054](app/app-builder.js#L1053-L1054)

**Risk:** When the QRCode library fails to load, the fallback calls `api.qrserver.com` with the full app URL. This sends business's app URLs to a third-party service. While the URLs are public, this is a privacy consideration and creates an external dependency.

**Fix:** Ensure the QRCode CDN library loads reliably (add fallback CDN), or generate a simple placeholder instead of calling an external API. Consider bundling `qrcode.js` locally.

### 7. [QA] Chart Period Default vs Active Button Mismatch

**File:** [dashboard.js:10](app/dashboard.js#L10), [dashboard.html:136-138](app/dashboard.html#L136-L138)

**Risk:** `currentPeriodDays` defaults to `30` but the 7D button has `class="active"` in the HTML. On load, the chart requests 30 days of data but the UI shows "7D" as selected.

**Fix:** Either change the default to `7` or set the 30D button as `active` in the HTML.

### 8. [CODE] `showToast()` Duplicated in App Builder

**File:** [app-builder.js:1347-1380](app/app-builder.js#L1347-L1380)

**Risk:** `app-builder.js` has its own `showToast()` implementation while `utils.js` likely provides `AppUtils.showToast()`. Having two implementations creates inconsistency in toast styling and behavior.

**Fix:** Use `AppUtils.showToast()` from `utils.js` consistently.

### 9. [DESIGN] Preview Panel Mobile Behavior Incomplete

**File:** [dashboard.css:1899-1925](app/dashboard.css#L1899-L1925)

**Risk:** At `< 1200px`, the preview column has `display: none` and requires `.open` class to show. But the toggle button logic in `showPreviewPanel()` doesn't add the `.open` class on mobile â€” the toggle button shows/hides using `display` directly, bypassing the CSS class.

**Fix:** Ensure the toggle button adds/removes `.open` class for mobile breakpoint, or unify the show/hide logic.

### 10. [QA] No Error Handling for Clipboard API Failure

**File:** [dashboard.js:436-441](app/dashboard.js#L436-L441)

**Risk:** `navigator.clipboard.writeText()` can fail (no HTTPS, permissions denied). The `.then()` handler updates button text but there's no `.catch()` â€” the copy silently fails with no user feedback.

**Fix:** Add `.catch()` that shows a toast or falls back to a text selection approach.

### 11. [CODE] Cache Bust Versions Not Aligned

**File:** [dashboard.html:449](app/dashboard.html#L449)

**Risk:** `dashboard.js?v=17` while `auth.js?v=4`, `sidebar.js?v=16`. These version numbers are manually managed and easy to forget. The session recap documents cache busting as critical, but the process is error-prone.

**Fix:** Consider a build step that auto-increments or uses file hashes. For now, document the current versions in the recap.
